# Проектные репозитории

## Главная идея

`Personal HQ` - это штаб: задачи, решения, исследования, handoff и карта проектов.

Отдельный проектный репозиторий нужен, когда появляется самостоятельный кодовый продукт: приложение, сайт, бот, сервер, библиотека, пайплайн или сервис.

## Когда оставить проект в `projects/`

Оставляй проект внутри штаба, если там в основном:

- идеи;
- задачи;
- бизнес-описание;
- исследования;
- контент;
- финансы без секретов;
- решения;
- ссылки;
- handoff.

## Когда создавать отдельный репозиторий

Создавай отдельный репозиторий, если появились:

- исходный код;
- зависимости;
- тесты;
- сборка;
- деплой;
- отдельные права доступа;
- отдельная команда;
- риск, что проект начнет засорять штаб.

## Как создать новый проектный репозиторий

1. Создай папку проекта вне штаба или новый GitHub-репозиторий.
2. Скопируй содержимое `templates/project-repo/` в корень нового репозитория.
3. Замени `PROJECT_NAME` в `README.md`.
4. Заполни назначение проекта в `AGENTS.md`.
5. Заполни `docs/architecture.md` и `docs/setup.md`.
6. Обнови `handoff/current.md` и `tasks/active/initial-setup.md`.
7. Сделай первый коммит.
8. В штабе добавь ссылку на новый репозиторий в соответствующий файл `projects/<project-id>/overview.md`.

## Команды, если репозиторий создается локально

```bash
mkdir my-project
cp -R path/to/personal-hq/templates/project-repo/. my-project/
cd my-project
git init -b main
git add .
git commit -m "Initialize project"
```

После создания приватного GitHub-репозитория:

```bash
git remote add origin https://github.com/<owner>/<repo>.git
git push -u origin main
```

## Как это экономит контекст

Codex не должен читать весь штаб для работы над кодовым проектом.

Для отдельного проектного репозитория Codex читает:

1. глобальный `~/.codex/AGENTS.md`;
2. проектный `AGENTS.md`;
3. `handoff/current.md`;
4. активный task-файл;
5. только нужные файлы проекта.

В штабе остается короткая карта проекта и ссылка на репозиторий. Это снижает лишнее чтение и помогает не смешивать контексты.
