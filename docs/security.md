# Безопасность

## Нельзя хранить в репозитории

- пароли;
- API-токены;
- cookie;
- приватные ключи;
- банковские данные;
- личные документы;
- дампы переписок с чувствительной информацией;
- реальные `.env` файлы.

## Можно хранить

- `.env.example` без реальных значений;
- инструкции по настройке;
- названия нужных переменных окружения;
- публичные ссылки;
- краткие выводы исследований.

## Проверки

Перед публикацией или `push` проверить:

```bash
git status
git diff --cached
```

Быстрая локальная проверка без внешних зависимостей:

```bash
rg -n --hidden --glob '!.git/**' --glob '!outputs/**' --glob '!work/**' \
  '(AKIA[0-9A-Z]{16}|gh[pousr]_[0-9A-Za-z_]{36,255}|sk-[0-9A-Za-z]{20,}|-----BEGIN (RSA |OPENSSH |EC |DSA )?PRIVATE KEY-----)' .
```

Если установлен `gitleaks`, перед `push` дополнительно выполнить:

```bash
gitleaks detect --source . --redact
```

## Текущий уровень защиты

- `.gitignore` исключает реальные `.env`, ключи, cookies, tokens, временные папки и локальное состояние Codex.
- В репозитории должны храниться только `.env.example` и инструкции без реальных значений.
- Секреты для GitHub, OpenAI, хостинга и ботов должны жить в менеджере секретов, переменных окружения или настройках CI/CD.
- Перед публикацией своей версии нужно удалить личный профиль, реальные проекты и все приватные handoff-файлы.
