# Git-процесс непрерывности

## Перед началом работы

```bash
git status
git pull
```

Если удаленный репозиторий еще не подключен, `git pull` пропускается до настройки GitHub.

## Во время работы

- Обновляй файл активной задачи в `tasks/active/`.
- Обновляй handoff-файл в `handoff/active/`.
- Если изменился статус, следующий шаг, текущий фокус или набор активных контекстов, обновляй `handoff/current.md`.

## После значимого этапа

```bash
git status
git add <файлы>
git commit -m "Краткое описание этапа"
git push
```

## При смене устройства или аккаунта

```bash
git pull
```

Затем открыть:

1. `handoff/current.md`;
2. нужные файлы из `handoff/active/`, перечисленные в активных контекстах;
3. нужные файлы из `tasks/active/`;
4. дочерние репозитории, если они указаны в активном контексте.

После этого новый чат должен понимать цель, текущее состояние и следующий шаг.
