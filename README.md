# Codex Personal HQ Template

Публичный шаблон личного штаба для работы с Codex, проектами, исследованиями, задачами и передачей контекста между устройствами, чатами и разрешенными аккаунтами.

Автор шаблона: Arthur.

## Главные принципы

- Главный источник правды: GitHub-репозиторий и Markdown-файлы.
- Рабочий язык: русский.
- Динамические факты проверяются с датой, временем и методом проверки.
- Каждая активная задача имеет `task-id`, файл задачи и handoff-файл.
- После значимого этапа обновляются task-файл, handoff и Git.
- Секреты, токены, cookie, пароли и приватные документы не хранятся в репозитории.

## Быстрый старт для нового чата

1. Открой корень репозитория.
2. Выполни `git pull`.
3. Прочитай `handoff/current.md`.
4. Открой связанный файл из `tasks/active/` и `handoff/active/`.
5. Создай видимый план Codex. Если план-трекер недоступен, веди чеклист в файле активной задачи.

## Как этим пользоваться

### Каждый новый чат

1. Начинай с `git status` и `git pull`.
2. Читай `handoff/current.md`.
3. Открывай указанную активную задачу и handoff.
4. Работай через видимый план.
5. После значимого этапа обновляй task-файл, handoff и Git.

### Каждый новый проект

- Если это идея, исследование, бизнес-направление или контент - держи его в `projects/`.
- Если появляется самостоятельный код, зависимости, сборка, тесты или деплой - создавай отдельный репозиторий из `templates/project-repo/`.
- В штабе оставляй короткую карту проекта, решения, ссылки и следующий шаг.

### Каждое исследование

- Для цен, сервисов, моделей, площадок и объявлений используй `docs/research-protocol.md`.
- Не пиши вывод как актуальный без даты, метода проверки и источника.

## Основные разделы

- `docs/` - правила, настройка Codex, Git-процесс, безопасность и оркестрирование.
- `tasks/` - входящие, активные, отложенные и архивные задачи.
- `handoff/` - состояние передачи задач между чатами и устройствами.
- `projects/` - проекты, продукты, бизнес-направления и примеры.
- `research/` - исследования с проверяемыми источниками.
- `templates/` - шаблоны для единообразной работы.
- `automation/` - будущая автоматизация настройки и проверок.

## Глобальные правила Codex

Резервная копия глобального `AGENTS.md` хранится в `templates/codex-global/AGENTS.md`.

На новом устройстве установи ее в:

- macOS/Linux: `~/.codex/AGENTS.md`
- Windows: `%USERPROFILE%\.codex\AGENTS.md`

## Новый кодовый проект

Если проект становится отдельным приложением, сайтом, ботом или сервисом, создай для него отдельный репозиторий из шаблона `templates/project-repo/`.

Подробно: `docs/project-repos.md`.

## Как использовать как шаблон

1. Создай новый репозиторий из этого шаблона.
2. Замени примеры в `projects/example-project/` на свои направления.
3. Обнови `templates/codex-global/AGENTS.md` под свои личные правила.
4. Обнови `handoff/current.md` и `tasks/active/initial-setup.md`.
5. Сделай первый commit от своего имени.
